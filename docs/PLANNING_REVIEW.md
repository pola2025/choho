# 초호펜션 기획서 점검 및 개선 제안

## 1. 점검 요약

기존 `PRD.md`와 `PLANNING.md`는 기술 스택(Next.js, Tailwind)과 전체적인 사이트 구조(메인, 객실, 카페, 소개)를 체계적으로 잘 정의하고 있습니다. 특히 **"네이버 예약 연동"**이라는 핵심 요구사항이 잘 반영되어 있습니다.

하지만 **"예약하려는 고객에게 기본 정보와 내용을 자세히 알려준다"**는 목표를 더 확실히 달성하기 위해, **콘텐츠 전달 방식**과 **사용자 경험(UX)** 측면에서 몇 가지 구체적인 개선이 필요합니다.

---

## 2. 주요 개선 제안

### 2.1 예약 전환 유도 (Call to Action) 개선

단순히 "예약하기" 버튼을 두는 것보다, 고객이 **"왜 네이버로 가야 하는지"** 이해하고 자연스럽게 넘어가도록 유도해야 합니다.

* **CTA 버튼 문구 변경**: "예약하기" → **"실시간 예약 현황 확인하기"** 또는 **"네이버에서 날짜 선택하기"**
  * *이유*: 고객은 '예약' 버튼을 누르면 바로 결제창이 뜰까 봐 주저할 수 있습니다. '현황 확인'은 부담 없이 클릭을 유도합니다.
* **플로팅 바 (Mobile Sticky Bar)**: 모바일 화면 하단에 항상 **"실시간 예약 확인"** 버튼을 고정 노출합니다.
  * *내용*: [날짜 선택] 버튼을 누르면 네이버 예약으로 연결.
* **신뢰 문구 추가**: 버튼 주변에 "네이버 예약을 통해 안전하고 간편하게 결제하세요" 문구 추가.

### 2.2 "기본 정보"의 디테일 강화 (Content Richness)

고객이 펜션 예약 전 궁금해하는 것은 매우 구체적입니다. 단순 텍스트 나열보다 **체크리스트**나 **Q&A** 형식이 효과적입니다.

* **"What's Provided" 체크리스트 (객실 상세)**:
  * 단순 "어메니티: 샴푸, 린스" 나열이 아닌, **"가져오셔야 할 것" vs "제공되는 것"**을 명확히 구분.
  * *예시*: "칫솔/치약은 제공되지 않으니 챙겨주세요.", "조미료는 위생상 제공되지 않습니다."
* **TMI (Too Much Information) 섹션**:
  * 주차장에서 객실까지의 거리/계단 유무 (짐 옮기기 편한지).
  * 가까운 편의점/마트 거리 (차량 5분 등).
  * 배달 음식 가능 여부.
  * *이유*: 이런 사소한 정보가 예약 결정에 결정적 역할을 합니다.

### 2.3 시각적 스토리텔링 (Visual Storytelling)

사진만 갤러리로 보여주는 것은 불친절할 수 있습니다.

* **포토 스토리 (Photo Story)**:
  * 단순 그리드 갤러리 대신, 사진 하단에 **캡션(설명)**을 달아주세요.
  * *예시*: 바베큐장 사진 밑에 "우천 시에도 이용 가능한 개별 바베큐장입니다."
* **공간감 전달**:
  * 평면도(Floor Plan) 이미지를 추가하여 방의 구조를 직관적으로 이해하게 합니다.

### 2.4 콘텐츠로서의 "공지사항" 업그레이드

`Notice`를 단순 공지사항이 아닌 **"초호 매거진"** 또는 **"여행 가이드"**로 확장합니다.

* **제안**: `Notice` → **`Journal`** 또는 **`Guide`**
* **내용**:
  * "초호펜션 200% 즐기는 법"
  * "근처 맛집 리스트 Best 5"
  * "사계절 초호의 풍경"
  * *이유*: 단순 공지는 재미없지만, 이런 콘텐츠는 고객이 사이트에 더 오래 머물게 하고 예약 욕구를 자극합니다.

## 3. 수정된 사이트맵 제안 (개선안 반영)

```diff
/                           # 홈페이지
├── /rooms                  # 객실 목록
│   ├── /rooms/[slug]       # 객실 상세
│   │   ├── (Section) 객실 소개 (포토 스토리)
│   │   ├── (Section) 상세 비품 리스트 (Checklist)
│   │   ├── (Section) 자주 묻는 질문 (FAQ)
│   │   └── (Floating) 실시간 예약 확인 버튼 (네이버 연동)
├── /cafe                   # 카페
├── /about                  # 소개
│   └── /about/journal      # (변경) 공지사항 -> 저널/가이드
├── /location               # 위치
│   └── (Add) 주변 편의시설/마트 안내
└── /faq                    # (신규) 통합 자주 묻는 질문
```

## 4. 결론

기획서의 기술적 기반은 탄탄합니다. 위에서 제안한 **"친절한 정보 제공"**과 **"네이버 예약으로의 자연스러운 연결"** 장치들을 추가한다면, 직접 결제 기능이 없더라도 높은 예약 전환율을 기대할 수 있습니다.
